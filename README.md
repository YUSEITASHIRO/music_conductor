# 🎼 Music Conductor

### *「音楽ファイルをアップロードして、両手を振るだけで、あなたが指揮者になれる。」*
> MediaPipe による手指リアルタイム認識を用いた、ブラウザだけで動く指揮体験Webアプリケーションです。

## 📖 制作背景 (Background)

子供の頃から、指揮者に憧れていました。
オーケストラの前に立ち、腕を振るうだけで音楽の流れが変わる——あの体験は、楽器の演奏とは全く異なる種類の「音楽への介入」です。

しかしプロの指揮者になる道は険しく、気軽に体験できる機会もほとんどありません。

そこで、**カメラとブラウザだけで再現できないか**と考えました。

お気に入りの音楽ファイルをアップロードし、画面の前で両手を動かすだけで、誰もがその曲の指揮者になれる——。
アプリのインストールも、特別なデバイスも不要。この手と、ブラウザだけで体験できる指揮を目指しました。

---

## 🎯 設計思想 (Design Philosophy)

### 1. 「右手でテンポ、左手で表情」——本物の指揮法を体で覚える
本物の指揮者は、右手でビートを刻みながら左手で強弱や表情をつけます。
本アプリでも**右手と左手に明確に異なる役割を割り当て**ることで、「楽器を弾く」のではなく「音楽を導く」という指揮固有の感覚を直感的に体験できるよう設計しました。

### 2. 「開く・握る」で音色が変わる
テンポと音量だけが指揮ではありません。
右手を**握れば（✊）** キレのあるクリアなサウンドに、**開けば（🖐）** 柔らかくマイルドな音色に変化します。
左手の開閉はリバーブ（残響）を制御し、ホールに響かせるか、ドライに締めるかを決めます。
「鳴らし方まで体で操る」という体験が、本作の最も指揮らしい要素です。

### 3. グレインプレイヤーによるピッチ維持
テンポを変えると音程も狂う——それでは指揮している気持ちになれません。
`Tone.js` の **`GrainPlayer`（グラニュラーシンセシス）** を採用し、音を細かい粒（グレイン）に分割して再合成することで、**テンポを上げ下げしても原曲の音程を保ったまま**自然にテンポ変化させることを実現しました。

### 4. 感度をプレイヤーが自分で調整できる設計
手の大きさ、動きのクセ、カメラとの距離は人によって異なります。
テンポ感度・フィルター感度・音量レンジ・リバーブ感度・スムージング強度のすべてをスライダーで調整できるため、誰がどんな環境で使っても「自分の動きに素直に反応する指揮棒」として機能します。

## 🎮 操作方法 (Controls)

| 手 | ジェスチャー | 効果 |
|----|------------|------|
| **右手** | 速く振る | テンポが上がる |
| **右手** | ゆっくり・止まる | テンポが落ちてスローになる |
| **右手** | 握る ✊ | クリア・キレのある音色 |
| **右手** | 開く 🖐 | マイルド・柔らかい音色 |
| **左手** | 高く上げる | 音量が上がる |
| **左手** | 低く下げる | 音量が下がる |
| **左手** | 開く 🖐 | リバーブ（残響）が増す |
| **左手** | 握る ✊ | ドライ（残響なし）になる |

## 🔧 工夫した点 (Implementation Notes)

### グレインプレイヤーによるピッチ維持
通常の `playbackRate` 変更では、テンポを上げると音程も上がってしまいます。
本アプリでは `Tone.GrainPlayer` を採用し、音を小さなグレイン（粒）に分割して再合成することで、**テンポを変えても原曲の音程を保った自然なタイムストレッチ**を実現しています。

### 左右反転カメラと手のラベル補正
ミラー表示のために Canvas を `scaleX(-1)` で反転しているため、MediaPipe が返す `label` の Left / Right が実際の表示と逆になります。コード内で `Left` ラベルを右手、`Right` ラベルを左手として扱うことで、**画面上の自然な対応関係を維持**しています。

### スムージングによる安定したテンポ制御
手の動きは瞬間的にブレが生じやすいため、右手の移動速度を直近Nフレーム分の移動平均で平滑化しています。さらに現在のplaybackRateとターゲット値を `0.9 : 0.1` の線形補間でなめらかに追従させることで、**急激なテンポ変化を防ぎながらも手の動きに即応する制御**を実現しています。

### 低周波数の底上げによる音切れ防止
フィルターのカットオフ周波数が下がりすぎると音が完全に消えてしまいます。
最低周波数を4000Hzに設定することで、**パー（開き）の状態でも音が聴こえ続ける**ようにし、演奏体験の連続性を確保しています。

## 📝 制作メモ (Retrospective)

「指揮者に憧れる」という個人的な動機が、このプロジェクトの出発点です。

最初に直面した壁は、**テンポを変えると音程が崩れる**という問題でした。`playbackRate` を直接操作するだけでは、速くするほど音が甲高くなり、「指揮している」というより「早送りしている」だけになってしまいます。`GrainPlayer` への切り替えが、この問題を解決する鍵でした。

次に時間を費やしたのが**スムージングの調整**です。手のブレをそのままパラメータに流すとテンポが暴れます。移動平均と線形補間を組み合わせても、係数の設定が少しでもズレると「遅延感のある操作」か「ガタガタのテンポ」かのどちらかになってしまいます。感度スライダーを設けたのは、「正解の値がひとつではない」という判断からでもあります。

右手でビートを刻み、左手で強弱をつけながら、好きな曲の指揮者になれる——その一点に、本作のすべてのこだわりが詰まっています。
